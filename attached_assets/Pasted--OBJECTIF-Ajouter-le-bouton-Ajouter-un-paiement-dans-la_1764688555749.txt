CONDITIONS D'AFFICHAGE DU BOUTON
Le bouton s'affiche UNIQUEMENT si ces 3 conditions sont réunies :
Type de facture : invoice.invoiceType === "supplier_invoice"
Les dépenses (expense) n'ont PAS ce bouton
Statut de paiement : invoice.paymentStatus !== "paid"
Les factures déjà soldées n'ont PAS ce bouton
Les factures "partial" ou "unpaid" ONT ce bouton
Permissions : invoice.userName === currentUser?.userName
Seul le créateur de la facture peut ajouter un paiement
COMPORTEMENT
Au clic sur le bouton :
Stopper la propagation de l'événement (e.stopPropagation()) pour éviter de déclencher d'autres actions
Définir la facture sélectionnée (setSelectedInvoice(invoice))
Ouvrir la modal (setIsAddPaymentModalOpen(true))
La modal AddPaymentModal gère :
Affichage du reste à payer
Formulaire de saisie (montant, date, type)
Validation
Soumission à l'API POST /api/payments
Fermeture et rafraîchissement

Variant : outline (bordure visible)

Couleurs : Texte et bordure teal-600 (couleur primaire de l'app)
Hover : Background teal-50
Icône : CreditCard de lucide-react
Texte : "Payer" (visible sur desktop, masqué sur mobile avec hidden sm:inline)
Responsive :
Sur mobile : Icône seule (💳)
Sur desktop : Icône + texte "Payer"
