## FORMAT EXPORT ZOHO - 27 COLONNES

### Headers CSV (ordre exact)
Entry Number,Expense Date,Expense Account,Paid Through,Vendor,Expense Description,Currency Code,Exchange Rate,Expense Amount,Tax Name,Tax Percentage,Is Inclusive Tax,Is Billable,Customer Name,Reference#,Mileage Rate,Distance,Start Odometer Reading,End Odometer Reading,Mileage Unit,Mileage Type,Expense Reference ID,Tax Type,Branch Name,Employee Email,CF.company,Project Name

### Paramètres CSV

- **Séparateur :** virgule (,)
- **Encodage :** UTF-8 avec BOM
- **Format date :** YYYY-MM-DD (ex: 2024-11-29)
- **Format montant :** décimales avec point (ex: 50000.00)
- **Guillemets :** autour des valeurs texte contenant virgules

### Mapping colonnes BDD → CSV Zoho

| N° | Colonne Zoho | Source BDD | Règle |
|----|--------------|------------|-------|
| 1 | Entry Number | - | Séquence 1, 2, 3... (numéro ligne export) |
| 2 | Expense Date | `invoices.date` | Format YYYY-MM-DD |
| 3 | Expense Account | `categories.account_name` | Nom compte comptable Zoho |
| 4 | Paid Through | `payment_methods_mapping.zoho_name` | Via LEFT JOIN payment_methods_mapping |
| 5 | Vendor | `suppliers.name` | Nom fournisseur |
| 6 | Expense Description | `invoices.description` | Description |
| 7 | Currency Code | Fixe | "XOF" |
| 8 | Exchange Rate | Fixe | 1 |
| 9 | Expense Amount | `invoices.amount_real_ttc` | Montant TTC réel comptable |
| 10 | Tax Name | Conditionnel | "TVA" si `vat_applicable = true`, vide sinon |
| 11 | Tax Percentage | Conditionnel | 18 si `vat_applicable = true`, vide sinon |
| 12 | Is Inclusive Tax | Conditionnel | true si `vat_applicable = true`, vide sinon |
| 13 | Is Billable | Fixe | False |
| 14 | Customer Name | Vide | (non applicable) |
| 15 | Reference# | `invoices.invoice_number` | DEP-MI-2511-001 |
| 16 | Mileage Rate | Vide | (non applicable) |
| 17 | Distance | Vide | (non applicable) |
| 18 | Start Odometer Reading | Vide | (non applicable) |
| 19 | End Odometer Reading | Vide | (non applicable) |
| 20 | Mileage Unit | Vide | (non applicable) |
| 21 | Mileage Type | Fixe | NonMileage |
| 22 | Expense Reference ID | Vide | (généré par Zoho) |
| 23 | Tax Type | Vide | (non applicable) |
| 24 | Branch Name | Vide | (non applicable) |
| 25 | Employee Email | Vide | (non applicable) |
| 26 | CF.company | Vide | (non applicable) |
| 27 | Project Name | `projects.name` | Nom du projet |

### Utilisation table payment_methods_mapping

**Jointure SQL :**
LEFT JOIN payment_methods_mapping ON invoices.payment_method = payment_methods_mapping.app_name

**Colonne "Paid Through" :**
- Si mapping trouvé → utiliser `payment_methods_mapping.zoho_name`
- Si mapping NULL (pas de correspondance) → utiliser `invoices.payment_method` tel quel (fallback)
- Logger un warning si fallback utilisé : "Mode paiement non mappé : [valeur]"

**Rappel mappings dans la table :**
- Wave Business → Banques - autres (Wave Business Principal)
- Wave Business Caisse → Banques - autres (Wave Business Caisse)
- Espèces → Caisse en monnaie nationale
- Perso remboursé → Banques - autres (Wave Business Principal)
- Carte bancaire → Banques en monnaie nationale

## RÈGLES DE FILTRAGE

### Critère de base obligatoire

**UNIQUEMENT les dépenses :**
WHERE invoice_type = 'expense'

Exclure toutes les Factures Fournisseurs (invoice_type = 'supplier_invoice').

### Filtrage par période (date pickers)

Interface avec 2 date pickers :
- **Date début** (label: "Du")
- **Date fin** (label: "Au")

**Si dates sélectionnées :**
WHERE date >= date_debut AND date <= date_fin

**Si dates vides :**
Exporter toutes les dépenses (pas de filtre date).

### Filtrage par utilisateur (4 boutons)

**Bouton 1 : Export Dépenses Michael**
- WHERE `user_id = [ID Michael]` AND `invoice_type = 'expense'`
- Nom fichier : `Depenses_Michael_YYYYMM.csv`

**Bouton 2 : Export Dépenses Marine**
- WHERE `user_id = [ID Marine]` AND `invoice_type = 'expense'`
- Nom fichier : `Depenses_Marine_YYYYMM.csv`

**Bouton 3 : Export Dépenses Fatou**
- WHERE `user_id = [ID Fatou]` AND `invoice_type = 'expense'`
- Nom fichier : `Depenses_Fatou_YYYYMM.csv`

**Bouton 4 : Export Toutes Dépenses**
- WHERE `invoice_type = 'expense'` (tous users confondus)
- Nom fichier : `Depenses_Toutes_YYYYMM.csv`

**Format nom fichier :**
- YYYYMM = année + mois de la date de fin sélectionnée
- Si pas de date fin sélectionnée → utiliser mois actuel
- Ex: date fin = 2024-11-30 → `Depenses_Michael_202411.csv`

### Tri des données

Trier par date croissante :
ORDER BY date ASC

## BACKEND - ROUTE EXPORT

### Route GET /api/exports/zoho/expenses

**Query params :**
- `user` : 'michael' | 'marine' | 'fatou' | 'all'
- `date_start` : YYYY-MM-DD (optionnel)
- `date_end` : YYYY-MM-DD (optionnel)

**Exemple requête :**
GET /api/exports/zoho/expenses?user=michael&date_start=2024-11-01&date_end=2024-11-30

### Logique backend

**Étape 1 : Requête SQL avec jointures**

Jointures nécessaires :
SELECT ...
FROM invoices
JOIN categories ON invoices.category_id = categories.id
JOIN suppliers ON invoices.supplier_id = suppliers.id
JOIN projects ON invoices.project_id = projects.id
LEFT JOIN payment_methods_mapping ON invoices.payment_method = payment_methods_mapping.app_name
WHERE invoice_type = 'expense'
AND [filtres utilisateur et dates]
ORDER BY date ASC

**Étape 2 : Construire filtres WHERE**

Filtres à appliquer :
- `invoice_type = 'expense'` (obligatoire)
- Si `user != 'all'` → ajouter `user_id = [ID]`
- Si `date_start` fourni → ajouter `date >= date_start`
- Si `date_end` fourni → ajouter `date <= date_end`

**Étape 3 : Mapper chaque ligne vers format Zoho**

Pour chaque facture récupérée :

1. **Entry Number** → index de la ligne + 1 (1, 2, 3...)
2. **Expense Date** → formater `date` en YYYY-MM-DD
3. **Expense Account** → `categories.account_name`
4. **Paid Through** → `payment_methods_mapping.zoho_name` (ou fallback `payment_method` si NULL)
5. **Vendor** → `suppliers.name`
6. **Expense Description** → `description`
7. **Currency Code** → "XOF"
8. **Exchange Rate** → 1
9. **Expense Amount** → `amount_real_ttc`
10. **Tax Name** → "TVA" si `vat_applicable = true`, sinon ""
11. **Tax Percentage** → 18 si `vat_applicable = true`, sinon ""
12. **Is Inclusive Tax** → true si `vat_applicable = true`, sinon ""
13. **Is Billable** → False
14. **Customer Name** → ""
15. **Reference#** → `invoice_number` (DEP-XX-YYMM-XXX)
16-20. **Mileage fields** → ""
21. **Mileage Type** → NonMileage
22-26. **Autres fields** → ""
27. **Project Name** → `projects.name`

**Étape 4 : Générer CSV**

Format CSV :
- Ligne 1 : Headers (27 colonnes séparées par virgule)
- Lignes suivantes : Données
- Encodage UTF-8 avec BOM (`\ufeff` au début)
- Échapper guillemets et virgules dans les descriptions

**Étape 5 : Retourner le fichier**

Headers HTTP :
Content-Type: text/csv; charset=utf-8
Content-Disposition: attachment; filename="Depenses_Michael_202411.csv"
