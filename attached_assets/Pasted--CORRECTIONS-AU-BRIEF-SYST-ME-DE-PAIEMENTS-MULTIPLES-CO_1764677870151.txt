### 4. SECTION "Migration des données existantes" → RÉÉCRIRE COMPLÈTEMENT

**RETIRER TOUTE LA SECTION ACTUELLE**

**REMPLACER PAR :**
9. Migration des données existantes
Stratégie de migration
Contexte :
Le champ invoice_type EXISTE DÉJÀ avec valeurs correctes ('expense' | 'supplier_invoice')
Les factures sont déjà classifiées correctement
Il faut uniquement créer la colonne payment_status et la table payments
Étape 1 : Ajouter colonne payment_status
ALTER TABLE invoices ADD COLUMN payment_status VARCHAR DEFAULT 'unpaid';
Étape 2 : Créer table payments
CREATE TABLE payments (
  id SERIAL PRIMARY KEY,
  invoice_id VARCHAR NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  amount_paid VARCHAR NOT NULL,
  payment_date VARCHAR NOT NULL,
  payment_type VARCHAR NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  created_by VARCHAR REFERENCES user_tokens(id)
);

CREATE INDEX idx_payments_invoice_id ON payments(invoice_id);
Étape 3 : Créer paiements automatiquement pour TOUTES les factures existantes
-- Créer un paiement à 100% pour chaque facture
INSERT INTO payments (invoice_id, amount_paid, payment_date, payment_type, created_by)
SELECT 
  id,
  COALESCE(amount_display_ttc, amount_ttc) as amount_paid,  -- Utiliser montant affiché si BRS
  COALESCE(invoice_date, created_at::date) as payment_date,
  payment_type,
  user_name  -- Note: Adapter si besoin de récupérer user_tokens.id
FROM invoices
WHERE invoice_date IS NOT NULL  -- Sécurité : ne traiter que factures valides
  AND payment_type IS NOT NULL;
Étape 4 : Mettre payment_status = 'paid' pour toutes les factures
-- Toutes les factures existantes sont considérées comme soldées
UPDATE invoices 
SET payment_status = 'paid'
WHERE id IN (SELECT invoice_id FROM payments);
Résultat migration :
✅ Toutes les factures existantes (dépenses ET factures fournisseur) ont payment_status = 'paid'
✅ Chaque facture a un paiement de 100% dans la table payments
✅ Date du paiement = date de la facture
✅ Type du paiement = type de règlement de la facture
✅ Le système de paiements multiples est prêt pour les nouvelles factures fournisseur
IMPORTANT :
Tester la migration sur une copie de la BDD d'abord
Backup de la BDD avant exécution
Vérifier que toutes les factures ont bien un paiement créé après migration
