# FORMULAIRE SOUMISSION FACTURES - LOGIQUE CONDITIONNELLE AVANCÉE

## CONTEXTE
Refonte complète du formulaire de soumission de factures avec logique conditionnelle intelligente pour réduire les erreurs de saisie et aligner sur le plan comptable Filtreplante. Structure BDD déjà en place avec table categories (14 entrées), champs is_regular_supplier sur suppliers, et nouveaux champs sur invoices.

## OBJECTIFS
1. Réorganiser le formulaire avec 14 champs dont 6 conditionnels
2. Implémenter le forcing automatique (TVA, Type de facture, Catégorie)
3. Gérer le calcul automatique HT/TTC avec TVA 18%
4. Gérer le calcul BRS pour prestations de services
5. Maintenir la compatibilité avec les 25 factures existantes (format ancien)
6. NE PAS TOUCHER AUX EXPORTS CSV (à faire plus tard dans une phase séparée)

## ORDRE ET LOGIQUE DES CHAMPS DU FORMULAIRE

### 1. Fournisseur (existant - dropdown)
- Source : table `suppliers`
- Afficher : `name`
- Récupérer en arrière-plan : `is_regular_supplier` (pour logique conditionnelle)
- Obligatoire

### 2. Montant TTC affiché (FCFA) (existant - remonté et renommé)
- Type : number input
- Label : "Montant TTC affiché (FCFA)"
- Stockage BDD : `amount_display_ttc`
- Obligatoire
- Validation : > 0
- Impact logique : déclenche forcing si >= 500000

### 3. Achat pour le stock ? (NOUVEAU - toggle)
- Type : toggle Oui/Non
- Défaut : Non
- Stockage BDD : `is_stock_purchase`
- Obligatoire

**LOGIQUE CONDITIONNELLE :**
- Si Oui → forcer automatiquement Catégorie = "Stock - achats de matériaux" (category_id = celui avec account_code '3210000000')
- Désactiver le dropdown catégorie (champ 4)
- Afficher un message : "ℹ️ Catégorie automatique : Stock - achats de matériaux"

### 4. Catégorie (modifié - nouveau dropdown avec BDD)
- Source : table `categories`
- Affichage dropdown : `app_name` (nom user-friendly)
- Stockage BDD : `category_id` (foreign key vers categories.id)
- Récupérer en arrière-plan : `account_name` et `account_code` (pour logique conditionnelle)
- Obligatoire
- Désactivé si "Achat pour le stock" = Oui

**LOGIQUE CONDITIONNELLE (forcing TVA) :**
- Si catégorie sélectionnée a `account_name = 'Réceptions'` (Restaurants) → forcer TVA = Non (champ 5)
- Si catégorie sélectionnée a `account_name = 'Fournitures non stockables - Energies'` (Essence) → forcer TVA = Non (champ 5)
- Dans ces cas, désactiver le toggle TVA et afficher message : "ℹ️ TVA automatiquement NON pour cette catégorie"

**LOGIQUE CONDITIONNELLE (forcing Type de facture) :**
- Si catégorie = 'Réceptions' (Restaurants) → forcer Type de facture = "Dépense" (champ 8)
- Si catégorie = 'Fournitures non stockables - Energies' (Essence) → forcer Type de facture = "Dépense" (champ 8)

### 5. TVA applicable (existant - toggle)
- Type : toggle Oui/Non
- Défaut : Non
- Stockage BDD : `has_vat` (champ existant)
- Obligatoire
- Peut être désactivé et forcé selon catégorie (voir champ 4)

**LOGIQUE CONDITIONNELLE (affichage Montant HT) :**
- Si TVA = Oui → afficher le champ 5b (Montant HT calculé)
- Si TVA = Non → masquer le champ 5b

**LOGIQUE CONDITIONNELLE (affichage BRS) :**
- Si TVA = Non ET catégorie avec `account_name = 'Achats d'études et prestations de services'` (Prestation de services)
  → Afficher le champ 6 (BRS mentionné)
- Sinon → masquer le champ 6

### 5b. Montant HT (NOUVEAU - conditionnel calculé)
- Type : number (readonly / calculé automatiquement)
- Label : "Montant HT (18% TVA)"
- Stockage BDD : `amount_ht`
- Affiché UNIQUEMENT si : TVA = Oui
- Calcul automatique en temps réel : `amount_display_ttc / 1.18`
- Format affichage : séparateur de milliers + " FCFA" (ex: "423 729 FCFA")

**RÈGLE DE CALCUL TVA :**
- Si TVA = Oui : `amount_ht = amount_display_ttc / 1.18`
- Si TVA = Non : `amount_ht = NULL` (ou `amount_ht = amount_display_ttc` selon préférence)

**PRÉSENTATION VISUELLE :**
Afficher 2 lignes sous le champ Montant TTC :
💰 Montant HT : 423 729 FCFA
💰 Montant TTC : 500 000 FCFA

### 6. BRS mentionné (NOUVEAU - conditionnel)
- Type : toggle Oui/Non
- Défaut : Non
- Label : "Montant total avec BRS mentionné"
- Stockage BDD : `has_brs`
- Affiché UNIQUEMENT si : TVA = Non ET Catégorie = "Prestation de services"

**LOGIQUE CONDITIONNELLE :**
- Si BRS = Non → afficher le champ 7 (Montant TTC réel)
- Si BRS = Oui → masquer le champ 7 + forcer Type de facture = "Facture Fournisseur" (champ 8)

### 7. Montant TTC réel (NOUVEAU - conditionnel calculé)
- Type : number input (readonly / calculé automatiquement)
- Label : "Montant TTC réel (comptable)"
- Stockage BDD : `amount_real_ttc`
- Affiché UNIQUEMENT si : BRS = Non (quand BRS est applicable)
- Calcul automatique en temps réel : `amount_display_ttc / 0.95`
- Format affichage : séparateur de milliers + " FCFA" (ex: "526 316 FCFA")

**RÈGLE DE CALCUL BRS :**
- Par défaut (si BRS non applicable ou BRS = Non) : `amount_real_ttc = amount_display_ttc`
- Si BRS = Oui : `amount_real_ttc = amount_display_ttc / 0.95`

### 8. Type de Facture (NOUVEAU - avec forcing complexe)
- Type : radio buttons (2 choix)
  - "Dépense"
  - "Facture Fournisseur"
- Stockage BDD : `invoice_type` (valeurs : 'expense' ou 'supplier_invoice')
- Obligatoire

**TEXTE DESCRIPTIF à afficher au-dessus des radio buttons :**

"**Facture Fournisseur :** Toute facture **supérieure à 500 000 FCFA** OU **à régler à une date ultérieure** OU provenant d'un **fournisseur régulier avec facture formelle** (ex. Europe Bache, CCS, Gaffari, etc.) OU provenant d'une prestation de service avec BRS applicable (uniquement prestation de service sans facture TVA)

**Dépense :** Toute dépense **inférieure à 500 000 FCFA**, **payée immédiatement** et ne provenant **pas d'un fournisseur régulier** avec facture formelle ou une prestation de service où BRS est applicable"

**LOGIQUE DE FORCING (ordre de priorité - vérifier dans cet ordre) :**

**CAS 1 - FORCER "Facture Fournisseur" (désactiver radio buttons) :**
1. Si `amount_display_ttc >= 500000` FCFA
2. OU si `supplier.is_regular_supplier = true`
3. OU si `has_brs = true`

**CAS 2 - FORCER "Dépense" (désactiver radio buttons) :**
4. Si catégorie avec `account_name = 'Réceptions'` (Restaurants)
5. OU si catégorie avec `account_name = 'Fournitures non stockables - Energies'` (Essence)

**CAS 3 - CHOIX LIBRE (activer radio buttons) :**
- Dans tous les autres cas où les conditions CAS 1 et CAS 2 sont fausses
- Laisser l'utilisateur choisir entre "Dépense" et "Facture Fournisseur"

**Message visuel :**
- Si forcé : afficher sous les radio buttons "⚠️ Type de facture déterminé automatiquement selon les critères définis"
- Si choix libre : ne rien afficher

### 9. Numéro de facture (NOUVEAU - conditionnel)

Logique de calcul en temps réel :
Calcul amount_ht (TVA 18%) :
useEffect(() => {
  if (hasVat) {
    setAmountHT(amountDisplayTTC / 1.18);
    setShowAmountHT(true);
  } else {
    setAmountHT(null); // ou amountDisplayTTC selon préférence
    setShowAmountHT(false);
  }
}, [amountDisplayTTC, hasVat]);
Calcul amount_real_ttc (BRS) :
useEffect(() => {
  if (hasBRS && showBRSField) {
    setAmountRealTTC(amountDisplayTTC / 0.95);
  } else {
    setAmountRealTTC(amountDisplayTTC);
  }
}, [amountDisplayTTC, hasBRS, showBRSField]);
Gestion affichage BRS :
useEffect(() => {
  const prestationServicesCategory = categories.find(
    c => c.account_name === "Achats d'études et prestations de services"
  );
  
  if (!hasVat && categoryId === prestationServicesCategory?.id) {
    setShowBRSField(true);
  } else {
    setShowBRSField(false);
    setHasBRS(false); // reset si plus applicable
  }
}, [hasVat, categoryId]);
Forcing catégorie si stock :
useEffect(() => {
  const stockCategory = categories.find(
    c => c.account_code === "3210000000"
  );
  
  if (isStockPurchase && stockCategory) {
    setCategoryId(stockCategory.id);
    setIsCategoryDisabled(true);
  } else {
    setIsCategoryDisabled(false);
  }
}, [isStockPurchase]);
Forcing TVA si catégorie Restaurant/Essence :
useEffect(() => {
  const category = categories.find(c => c.id === categoryId);
  
  if (category && (
    category.account_name === 'Réceptions' || 
    category.account_name === 'Fournitures non stockables - Energies'
  )) {
    setHasVat(false);
    setIsTVADisabled(true);
  } else {
    setIsTVADisabled(false);
  }
}, [categoryId]);
Forcing Type de facture (logique complexe) :
useEffect(() => {
  const category = categories.find(c => c.id === categoryId);
  
  // CAS 1: Forcer Facture Fournisseur
  if (
    amountDisplayTTC >= 500000 ||
    selectedSupplier?.is_regular_supplier ||
    hasBRS
  ) {
    setInvoiceType('supplier_invoice');
    setForcedInvoiceType('supplier_invoice');
    setIsInvoiceTypeDisabled(true);
    return;
  }
  
  // CAS 2: Forcer Dépense
  if (category && (
    category.account_name === 'Réceptions' ||
    category.account_name === 'Fournitures non stockables - Energies'
  )) {
    setInvoiceType('expense');
    setForcedInvoiceType('expense');
    setIsInvoiceTypeDisabled(true);
    return;
  }
  
  // CAS 3: Choix libre
  setForcedInvoiceType(null);
  setIsInvoiceTypeDisabled(false);
}, [amountDisplayTTC, selectedSupplier, hasBRS, categoryId]);
Affichage conditionnel Numéro de facture :
useEffect(() => {

    setShowInvoiceNumberField(true);
