## PARTIE 1 : LISTE DES FACTURES

### Colonnes à afficher

**Colonnes de base :**
- Date
- Fournisseur
- N° facture (si invoice_type = 'supplier_invoice')
- Catégorie (app_name)
- Projet
- Description (tronquée à 50 caractères)
- Montant TTC
- Montant HT (si vat_applicable = true)
- Actions (Voir, Modifier, Supprimer)

**Badges et indicateurs :**
- Badge Type : "Dépense" (bleu) ou "Facture Fournisseur" (vert)
- Étiquette BRS : "BRS 5%" (gris) si has_brs = true
- Icône Stock : 📦 si is_stock_purchase = true

### Filtres

Barre de filtres au-dessus du tableau :
1. **Type de facture** : Dropdown "Tous" / "Dépense" / "Facture Fournisseur"
2. **Catégorie** : Dropdown avec les 14 catégories + "Toutes"
3. **Avec BRS uniquement** : Checkbox
4. **Achats stock uniquement** : Checkbox

### Tri

Par défaut : date décroissante (plus récent d'abord)
Permettre tri sur : Date, Fournisseur, Montant
Direction : croissant/décroissant (icône flèche)

### Backend : Route GET /api/invoices avec filtres

Adapter la route pour accepter query params :
- `type` : 'expense' | 'supplier_invoice' | 'all'
- `category_id` : integer | 'all'
- `has_brs` : boolean
- `is_stock_purchase` : boolean
- `sort_by` : 'date' | 'supplier' | 'amount'
- `sort_order` : 'asc' | 'desc'

Filtrer et trier côté serveur avec SQL WHERE et ORDER BY.

### Responsive mobile

**Vue mobile (< 768px) :**
Cards avec :
- Ligne 1 : Date + Badge Type
- Ligne 2 : Fournisseur + N° facture (si existe)
- Ligne 3 : Catégorie + Indicateurs (BRS, Stock)
- Ligne 4 : Montant TTC + HT (si TVA)
- Ligne 5 : Description tronquée
- Ligne 6 : Boutons actions

**Vue desktop :**
Tableau avec toutes les colonnes.

## PARTIE 2 : VUE DÉTAIL EN MODALE

Quand l'utilisateur clique sur "Voir", ouvrir une modale avec :

**Header :**
- Titre : "Facture du [date]" + Badge Type
- Bouton fermer (X)

**Contenu (sections) :**

**Section 1 : Informations générales**
- Date
- Fournisseur
- N° facture (si Facture Fournisseur)
- Projet
- Description complète

**Section 2 : Informations financières**
- Catégorie (app_name + account_name entre parenthèses)
- Montant TTC à régler
- Montant HT (si vat_applicable = true)
- Facture avec TVA (18%) : Oui/Non
- BRS applicable : Oui/Non
  - Si Oui → afficher "Retenue BRS : X FCFA sur Y FCFA" (calcul : 5% de amount_real_ttc)
- Achat pour le stock : Oui/Non
- Type de règlement

**Section 3 : Justificatif**
- Photo de la facture en grand format
- Possibilité de zoom ou ouvrir dans nouvel onglet

**Footer :**
- Bouton "Modifier"
- Bouton "Supprimer" (avec confirmation)
- Bouton "Fermer"

## PARTIE 3 : ÉDITION - RÉUTILISATION COMPOSANT

### Stratégie

Réutiliser le composant **InvoiceForm** créé en Phase 2 avec un prop `mode`.

**Props à ajouter :**
mode: 'create' | 'edit'
invoiceData: object (si mode = 'edit')
onSuccess: callback

**Comportement en mode 'edit' :**
1. Pré-remplir tous les 14 champs avec invoiceData
2. Appliquer la même logique conditionnelle (7 useEffect)
3. Changer endpoint : PUT /api/invoices/:id
4. Validation backend identique à POST

**Ouverture du formulaire édition :**
Depuis la modale détail, cliquer "Modifier" → ouvrir InvoiceForm en mode 'edit' (peut être dans une nouvelle modale ou remplacer le contenu).

### Backend : Route PUT /api/invoices/:id

Créer ou adapter cette route avec :
- Toutes les validations de POST /api/invoices (Phase 1)
- Recalcul amount_ht et amount_real_ttc
- Vérification règles de forcing
- Retourner la facture mise à jour
