## PARTIE 1 : LISTE DES FACTURES (Tableau/Cards)

### Colonnes/Informations à afficher

**Affichage existant à conserver :**
- Date
- Fournisseur
- Projet
- Description (tronquée si longue)
- Montant
- Photo (icône/lien)
- Actions (voir, modifier, supprimer)

**NOUVELLES colonnes à ajouter :**

1. **Type de facture** (nouveau)
   - Badge visuel : 
     - "Dépense" → badge bleu
     - "Facture Fournisseur" → badge vert
   - Si invoice_type IS NULL (anciennes factures) → ne pas afficher de badge

2. **N° facture** (nouveau)
   - Afficher si invoice_type = 'supplier_invoice'
   - Format : petit texte gris sous le fournisseur
   - Si NULL → ne rien afficher

3. **Catégorie** (modifié)
   - Afficher app_name depuis la table categories
   - Si category_id IS NULL (anciennes factures) → afficher l'ancien champ category (texte libre)
   - Si les deux sont NULL → afficher "Non définie"

4. **Montant HT** (nouveau - conditionnel)
   - Afficher uniquement si vat_applicable = true ET amount_ht IS NOT NULL
   - Format : "HT: XXX FCFA" en petit sous le montant TTC
   - Si NULL ou vat_applicable = false → ne pas afficher

5. **BRS** (nouveau - indicateur)
   - Afficher une petite étiquette "BRS 5%" si has_brs = true
   - Couleur neutre (gris/bleu)
   - Si false ou NULL → ne pas afficher

6. **Stock** (nouveau - indicateur)
   - Afficher une petite icône 📦 si is_stock_purchase = true
   - Avec tooltip "Achat pour le stock"
   - Si false ou NULL → ne pas afficher

### Adaptation responsive mobile

**Vue mobile (< 768px) :**
Afficher en cards au lieu de tableau, avec :
- Ligne 1 : Date + Badge Type
- Ligne 2 : Fournisseur + N° facture (si existe)
- Ligne 3 : Catégorie + Icônes (BRS, Stock)
- Ligne 4 : Montant TTC + Montant HT (si TVA)
- Ligne 5 : Description tronquée
- Ligne 6 : Actions

**Vue desktop :**
Tableau avec toutes les colonnes.

### Tri et filtres (optionnel - si facile)

Permettre de :
- Trier par date (défaut : plus récent d'abord)
- Filtrer par Type de facture (Tous / Dépense / Facture Fournisseur)
- Filtrer par Catégorie (dropdown avec les 14 catégories)

Si trop complexe, laisser pour plus tard.

## PARTIE 2 : VUE DÉTAIL D'UNE FACTURE

### Affichage en lecture seule

Quand l'utilisateur clique sur "Voir", afficher une modale ou page avec TOUTES les informations :

**Section 1 : Informations générales**
- Date
- Fournisseur (nom complet)
- Projet (nom complet)
- Description complète

**Section 2 : Informations financières**
- Type de facture (badge)
- N° facture (si Facture Fournisseur)
- Catégorie (app_name + account_name entre parenthèses)
- Montant TTC à régler
- Montant HT (si TVA applicable)
- TVA applicable (Oui/Non)
- BRS applicable (Oui/Non) - si Oui, afficher calcul "Retenue BRS : X FCFA sur Y FCFA"
- Achat pour le stock (Oui/Non)
- Type de règlement
- Photo de la facture (affichage grand format)

**Gestion rétrocompatibilité :**
- Si invoice_type IS NULL → ne pas afficher "Type de facture"
- Si category_id IS NULL → afficher "Catégorie : [ancien champ category]"
- Si amount_ht IS NULL → ne pas afficher ligne Montant HT
- Afficher uniquement les champs qui ont des valeurs

**Boutons d'action :**
- Modifier (ouvre formulaire édition)
- Supprimer (avec confirmation)
- Fermer

## PARTIE 3 : ÉDITION D'UNE FACTURE

### Comportement attendu

Quand l'utilisateur clique sur "Modifier" :

**Pour les NOUVELLES factures (avec category_id NOT NULL) :**
- Ouvrir le formulaire complet avec les 14 champs
- Pré-remplir tous les champs avec les valeurs existantes
- Permettre la modification de tous les champs
- Appliquer la même logique conditionnelle que lors de la création
- Validation backend identique

**Pour les ANCIENNES factures (category_id IS NULL) :**
Deux options au choix :

**Option A (plus simple) :**
- Afficher un message : "Cette facture utilise l'ancien format. Modifiez-la depuis l'interface standard."
- Rediriger vers l'ancien formulaire d'édition (si existe)

**Option B (plus complexe) :**
- Permettre l'édition mais forcer la migration vers le nouveau format
- Afficher un message : "⚠️ Cette facture sera mise à jour vers le nouveau format lors de la sauvegarde"
- Forcer à sélectionner une catégorie depuis les 14 disponibles
- Compléter les champs manquants (Type de facture, etc.)

Choisir l'option la plus simple pour cette phase. On peut améliorer plus tard.
