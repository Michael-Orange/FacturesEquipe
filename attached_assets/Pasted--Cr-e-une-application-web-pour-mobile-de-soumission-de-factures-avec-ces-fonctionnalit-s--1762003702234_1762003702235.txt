



Crée une application web (pour mobile) de soumission de factures avec ces fonctionnalités :





ACCÈS :

- 3 URLs différentes avec des tokens uniques (un par membre d'équipe)

- Quand quelqu'un visite son URL, il voit directement le formulaire avec son nom pré-rempli

- Si le token est invalide, afficher "Accès refusé"



FORMULAIRE À CRÉER :

- Nom (pré-rempli automatiquement selon le token) : 3 choix Michael, Marine, Fatou

- Date de la facture : Date préremplie par défaut à la date du jour

- Fournisseur : Liste déroulante (fichier CSVFournisseur) + possibilité de créer un fournisseur si besoin. Lors de la création vérifier qu'il n'existe aps avec une orthographe proche et confirmer avec le user la création d'un nouveau fournisseur. Voir section FOURNISSEUR

- Catégorie (menu déroulant : Restauration, Essence, Fourniture Matériaux, Achats Prestas, Transport de matériel, Hébergement, Telephone/Internet, Autre)

- Montant TTC : champs libre (chiffre uniquement)

- TVA applicable : Oui / Non (champs à ne pas afficher si Restauration)

- Montant HT (optionnel, uniquement si TVA Oui)

- Description

- Type de réglement (Wave, Espèces) ; Ajouter uniquement pour Marine et Michael les choix en plus (Espèces remboursés par Wave Business)

- Projets : (uploader un fichier CSV Projets)

- Upload de fichier (PDF ou image de la facture)



Pour els upload projets et fournisseurs, c'est juste à la création de l'app. Ensuite ces données doivent etre dans la BDD de replit et plus utiliser les CSV. 



FOURNISSEURS : 

GESTION DES FOURNISSEURS - Besoin fonctionnel

Champ "Fournisseur" dans le formulaire

1. Champ de recherche intelligent





Afficher un champ de recherche avec placeholder "🔍 Rechercher un fournisseur..."



Au fur et à mesure de la saisie, afficher les suggestions correspondantes (maximum 10 résultats)



La recherche doit être tolérante aux fautes d'orthographe (fuzzy search)



Quand l'utilisateur sélectionne un fournisseur, le champ se remplit avec le nom exact

2. Raccourcis rapides - Section "Mes derniers"





Afficher les 5 derniers fournisseurs utilisés par l'utilisateur connecté



Basé sur l'historique personnel de ses factures soumises



Cliquable pour sélection rapide



Si l'utilisateur n'a pas d'historique, ne pas afficher cette section

3. Raccourcis rapides - Section "Plus gros volumes"





Afficher les 5 fournisseurs avec les plus gros montants dans la colonne "total" du CSV



Ne pas afficher les montants



Cliquable pour sélection rapide



Cette liste est la même pour tous les utilisateurs

4. Bouton "Nouveau fournisseur"





Bouton distinct : [+ Nouveau fournisseur]



Au clic, ouvrir un champ de saisie libre



Quand l'utilisateur tape le nom du nouveau fournisseur :





Vérifier automatiquement s'il existe un fournisseur avec un nom similaire dans le CSV



Si oui, afficher une popup de confirmation :



"⚠️ Un fournisseur similaire existe déjà : [Nom du fournisseur existant]. Voulez-vous utiliser ce fournisseur existant ou créer un nouveau ?"



Proposer 2 boutons : [Utiliser l'existant] et [Créer quand même]



Si l'utilisateur confirme la création, ajouter le nouveau fournisseur à la base de données (pas au CSV)

5. Comportement mobile





L'interface doit être optimisée pour mobile (écran tactile)



Les zones cliquables doivent être suffisamment grandes (minimum 44px de hauteur)



Le champ de recherche doit avoir le focus automatique quand on arrive sur le formulaire

6. Design





Les sections "Mes derniers" et "Plus gros volumes" doivent être visuellement distinctes



PROJETS 



MENU DÉROULANT PROJETS - Besoin fonctionnel

Champ "Projet/Opération" dans le formulaire

1. Type de champ





Utiliser un menu déroulant natif HTML (<select>)



Optimisé pour mobile (interface tactile native du système)

2. Organisation en 2 groupes

Le menu déroulant doit être organisé en 2 sections distinctes :

Groupe 1 : "🟢 PROJETS ACTIFS 2025"





Afficher tous les projets dont le numéro commence par "2025-"



Trier par numéro décroissant (2025-37 avant 2025-35, etc.)

Groupe 2 : "📋 PROJETS 2024 ET ANTÉRIEURS"





Afficher tous les autres projets (2024-, 2012-, etc.)



Trier par numéro décroissant dans chaque année



L'option "2024-10 - Divers - Non classé" doit toujours être en dernier de ce groupe

3. Format d'affichage





Chaque option doit afficher : Numéro - Nom du projet



Exemples :





2025-37 - HAA Kolda



2025-35 - MPP



2024-20 - STEP Rosso

4. Valeur par défaut





Le champ doit être vide par défaut (pas de sélection)



Afficher un placeholder : "Sélectionner un projet..."
